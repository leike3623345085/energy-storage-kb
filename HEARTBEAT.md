# HEARTBEAT.md - 会话启动与记忆管理

> 每次会话开始前的强制检查清单
> 确保保存的记忆被真正使用

---

## 会话启动清单（每次开始对话必须执行）

### 第一步：加载核心身份
- [ ] 读取 `SOUL.md` - 确认人格、风格、边界
- [ ] 读取 `USER.md` - 确认用户偏好、沟通方式

### 第二步：加载近期上下文
- [ ] 读取 `memory/YYYY-MM-DD.md` - 今日日志（当天+昨天）
- [ ] 【关键】运行 `memory/context_loader.py` - 生成活跃项目摘要

### 第三步：生成记忆摘要
执行以下命令，准备会话上下文：
```bash
cd /root/.openclaw/workspace && python3 memory/context_loader.py
```

预期输出示例：
```
📁 进行中的项目: energy-storage, bank-workflow
⚠️ 已记录教训: 2 项（储能爬虫修复、邮件配置）
📝 近7天对话归档: 3 天
```

## 任务执行前检查清单（执行任何任务前）

当用户请求任务时，必须检查：

### 1. 相似任务检查
```
用户说："帮我写个爬虫"
→ memory_search("爬虫 python 储能") 
→ 是否做过？→ 复用/改进/新建
```

### 2. 相关项目检查
```
用户说："银行工作流"
→ 读取 memory/system/bank-workflow.md
→ 检查是否有进行中的相关任务
```

### 3. 经验教训检查
```
用户说："部署储能系统"
→ 读取 memory/lessons/energy-storage.md
→ 避免重复犯错
```

### 4. 用户偏好检查
```
用户说："生成报告"
→ 检查 USER.md 中 "偏好：简洁/详细"
→ 检查之前报告的格式偏好
```

## 关键词触发规则

以下关键词必须触发记忆搜索：

| 关键词类别 | 示例 | 搜索范围 |
|-----------|------|---------|
| 时间回溯 | "之前、上次、以前、记得" | 全部记忆 |
| 项目引用 | "储能、银行、工作流、爬虫" | system/ 目录 |

| 任务类型 | "报告、文档、配置、部署" | 相关项目文档 |

## 记忆使用验证

每次使用记忆后，验证是否有效：

```
使用了记忆 → 记录到当日日志
├── 使用了什么记忆？（文件路径）
├── 解决了什么问题？
└── 是否避免了重复工作？
```

---

## 失效检测

如果发现以下情况，说明记忆机制失效：

- ❌ 用户重复问同一个问题
- ❌ 重复犯同一个错误
- ❌ 生成了与之前冲突的内容
- ❌ 忽略了用户的明确偏好

**失效处理：**
1. 立即道歉并纠正
2. 检查为何记忆未被唤起
3. 更新记忆文件或触发机制
4. 记录到 lessons/ 避免再犯
