## 🚨 经验教训（2026-03-06）

### 问题：日报生成任务执行失败
**时间**：2026-03-06 18:01
**表现**：定时任务状态显示"ok"，但实际报告未生成，邮件未发送
**原因**：AGENT（我）收到"执行脚本"指令后，只解释了脚本功能，未实际调用 exec 工具执行
**性质**：执行能力问题，非配置问题

### 根本原因
Cron 任务使用 `payload.kind: "agentTurn"`，AGENT 把脚本执行任务理解为"解释说明任务"，输出文字分析而非执行命令。

### 解决方案
1. **修改任务指令**：明确标注"【重要：必须使用 exec 工具实际执行命令】"
2. **区分任务类型**：
   - 信息查询类 → 直接回答
   - 脚本执行类 → 必须使用 exec/shell 工具
3. **双重确认**：看到 "python3 xxx.py" 指令时，先执行再反馈
