# MEMORY.md - 储能行业监控系统

## 系统概述
24小时持续监控储能行业最新动态，自动生成日报/周报并邮件推送。

## 核心定位（2026-03-07更新）
> **新型电力系统的"稳定器"** — 从根本上解决风电/光伏的"间歇性"，实现削峰填谷

### 战略定位
- **根本使命**：解决风光间歇性，实现削峰填谷
- **战略角色**：新型电力系统的"稳定器"、新能源的刚需配套
- **支撑目标**：服务"双碳"战略

### 技术覆盖领域
| 领域 | 关键技术/方向 |
|------|---------------|
| **储能系统集成** | 压缩空气储能、飞轮储能、储热/储冷、氢能储能 |
| **电池多元化** | 锂电池(LFP)、钠离子电池、液流电池(全钒/锌溴/铁铬)、固态/半固态电池 |
| **PCS与电力电子** | 储能变流器、构网型储能(grid-forming)、高压直挂/级联式储能 |
| **消防与安全** | pack级消防、电池热失控防控、全氟己酮等气体消防 |
| **智能运维** | BMS/EMS、数字孪生、储能云平台、远程监控 |
| **虚拟电厂** | VPP聚合、源网荷储一体化、车网互动(V2G)、微电网 |

### 政策关注重点
- **标准统一**：储能并网标准、消防安全标准
- **调度机制**：储能调度规则、电力现货市场准入
- **商业模式**：容量租赁、辅助服务收益、电力交易

### 日报/周报聚焦方向
1. 新型电力系统建设进展
2. 风光大基地配套储能项目
3. 储能系统集成技术突破（压缩空气/飞轮/氢能等）
4. 电池多元化产业化动态（钠电/液流/固态）
5. 构网型储能技术进展
6. 储能消防与安全标准
7. 虚拟电厂商业模式与政策
8. 电力市场化改革对储能的影响

### 定时任务超时自动处理（2026-03-14）
**问题**：3个定时任务连续超时（系统巡检/日记保存/工作日志保存）
**根因**：
1. self_healing.py 包含多个长时间子命令，累计超时
2. diary_save.py 调用 `openclaw sessions history` CLI 命令卡住
3. 超时设置不合理（60秒/300秒不足）
**修复**：
1. 创建 self_healing_fast.py - 轻量版只检查不修复，移除耗时操作
2. 创建 diary_save_simple.py - 直接读取文件，不依赖 CLI
3. 调整超时：巡检60秒、日记30秒、工作日志120秒
4. 更新3个定时任务配置使用新脚本
**验证**：手动测试新脚本，执行时间 < 5秒

### 对话会话抢占（2026-03-11）
**问题**：对话说到一半突然无响应，心跳消息抢占主会话
**根因**：OpenClaw心跳每30秒触发一次，与正常对话共用会话导致抢占
**修复**：1.创建独立守护进程daemon_guardian.py 2.后台静默执行巡检 3.心跳与对话隔离
**验证**：对话稳定性已恢复，无中断现象
**详情**：见 memory/2026-03-11.md

### 任务执行中断（2026-03-11）
**问题**：Cron任务执行时Agent工具调用验证错误，导致邮件发送步骤被跳过
**根因**：Agent在isolated session中工具调用遇到验证错误，任务提前结束
**修复**：1.更新定时任务配置添加exec强制提示 2.部署自愈监控机制 3.增加多重保障
**验证**：模拟日报缺失场景，2秒内自动检测并修复，邮件发送成功

**性质**：执行能力问题，非配置问题

### 根本原因
Cron 任务使用 `payload.kind: "agentTurn"`，AGENT 把脚本执行任务理解为"解释说明任务"，输出文字分析而非执行命令。

### 解决方案
1. **修改任务指令**：明确标注"【重要：必须使用 exec 工具实际执行命令】"
2. **区分任务类型**：
   - 信息查询类 → 直接回答
   - 脚本执行类 → 必须使用 exec/shell 工具
3. **双重确认**：看到 "python3 xxx.py" 指令时，先执行再反馈

## 🎯 系统架构规范（2026-03-21 确立）

### Harness Engineering 强制规范

> **以后所有工作流都必须复用 Harness Engineering 架构**

#### 核心原则
- **"人类掌舵，智能体执行"** — Humans steer. Agents execute.
- **"自主权 = f(背压)"** — 约束和反馈越完善，AI 自主权越大

#### 新建工作流必须包含的组件

| 组件 | 必需 | 说明 |
|------|------|------|
| **Guardrails** | ✅ | 飞行前检查、输出验证 |
| **Feedback Loop** | ✅ | 错误分类、自动修复、规则学习 |
| **Progressive Disclosure** | ✅ | 分层上下文加载 |
| **Agent Runner** | ✅ | 总控协调 |

#### 工作流模板

所有新工作流必须基于以下模板：

```python
# 导入 Harness 组件
sys.path.insert(0, str(Path(__file__).parent / 'harness'))
from agent_runner import AgentRunner
from guardrails import GuardrailsSystem
from feedback_loop import FeedbackLoop
from progressive_context import ProgressiveDisclosure

# 初始化
runner = AgentRunner()

# 执行工作流
result = runner.execute_workflow('workflow_name', **params)
```

#### 禁止事项

- ❌ 直接调用 AI 生成而不经过 Guardrails 验证
- ❌ 忽略错误处理，不接入 Feedback Loop
- ❌ 一次性加载所有上下文，不使用 Progressive Disclosure
- ❌ 重复造轮子，不复用 harness/ 已有组件
