## 一、系统概述

### 1.1 系统定位

**新型电力系统的"稳定器"** —— 24小时持续监控储能行业动态，自动生成分析报告。

### 1.2 核心能力

| 能力 | 说明 |
|------|------|
| **数据采集** | 网站爬虫 + API监控 + 搜索聚合 |
| **智能分析** | MECE热点提炼 + SWOT速览 + 产业链动态 |
| **自动报告** | 日报(18:05) + 深度分析(18:10) + 周报(周一9:00) |
| **多平台同步** | 邮件 + IMA笔记 + GitHub备份 + 向量知识库 |
| **自愈系统** | 故障自动检测 → 分类 → 修复 → 学习 |

## 二、核心架构

### 2.1 Harness Engineering 三层架构

```
┌─────────────────────────────────────────────────────────────┐
│  Guardrails (护栏)                                          │
│  ├─ 数据量检查 (爬虫≥20条, 新闻≥10条)                        │
│  ├─ 格式验证 (必须包含4个章节)                                │
│  └─ 超时检查 (日报≤600秒)                                    │
├─────────────────────────────────────────────────────────────┤
│  Progressive Disclosure (渐进式披露)                         │
│  ├─ SIMPLE: 当日数据 (日报生成)                              │
│  ├─ COMPLEX: 7天数据 (深度分析)                              │
│  └─ DIAGNOSTIC: 日志+历史 (故障排查)                         │
├─────────────────────────────────────────────────────────────┤
│  Feedback Loop (反馈循环)                                    │
│  ├─ 错误分类 (E001数据不足, E002格式错误...)                  │
│  ├─ 自动修复 (5种策略)                                        │
│  └─ 规则学习 (更新 learned_rules.json)                       │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 记忆系统 v2 (OpenViking 思想)

| 层级 | 内容 | Token | 用途 |
|------|------|-------|------|
| **L0** | 一句话摘要 | ~100 | 快速判断相关性 |
| **L1** | 章节概览+关键点 | ~2000 | 规划阶段决策 |
| **L2** | 原始完整内容 | 按需 | 深入分析时读取 |

## 三、定时任务清单

### 3.1 数据采集类

| 任务 | 频率 | 时间 | 输出 |
|------|------|------|------|
| 网站爬虫 | 每4小时 | 00/04/08/12/16/20 | `crawler/*.json` |

| 晚班搜索 | 每天1次 | 20:00 | `news/*.json` |

### 3.2 报告生成类 (18:00密集时段)

| 时间 | 任务 | 动作 |
|------|------|------|
| 18:05 | 日报生成 | MECE→SWOT→产业链→邮件发送 |
| 18:10 | 深度分析 | 7天数据+趋势分析→邮件发送 |
| 18:20 | 自愈检查 | 检查日报状态，失败则修复 |
| 18:25 | 知识库索引 | 日报向量化入库 |
| 18:30 | 知识库索引 | 深度分析向量化入库 |
| 18:35 | 邮件补发检查 | 如未发送则补发 |
| 18:50 | 最终检查 | 通知到kimi-claw |
| 19:00 | 深度分析后自愈 | 检查深度分析状态 |

### 3.3 维护类

| 任务 | 频率 | 时间 | 说明 |
|------|------|------|------|
| IMA同步 | 每4小时 | 随爬虫 | 同步到「全部笔记」 |
| GitHub备份 | 每天 | 23:50 | 增量提交数据文件 |
| 数据清理 | 每天 | 02:00 | 保留7天数据 |
| 记忆索引构建 | 每天 | 09:30 | 重建L0/L1摘要 |
| 每周回顾 | 周日 | 10:00 | 提炼经验教训 |

## 五、故障处理

### 5.1 常见故障类型

| 代码 | 故障描述 | 自动修复策略 |
|------|----------|--------------|
| E001 | 爬虫数据不足(<20条) | 重试3次→调整阈值→通知 |
| E002 | 报告格式错误 | 检查模板→重新生成 |
| E003 | 邮件发送失败 | 重试→换备用SMTP→通知 |
| E004 | AI服务过载 | 指数退避重试→记录规则 |
| E005 | 超时(>600秒) | 增加超时时间→减少并发 |

### 5.2 自愈系统触发条件

```
日报未生成
    ↓
18:20 自愈检查 → 尝试修复 → 成功→结束
    ↓ 失败
18:35 补发检查 → 尝试补发 → 成功→结束
    ↓ 失败
18:50 最终检查 → 通知到这里
```

### 5.3 手动修复命令

```bash
# 重新生成日报
cd energy_storage && python3 generate_report_harness_v3.py

# 重新发送邮件
cd energy_storage && python3 send_email.py --date $(date +%Y%m%d)

# 手动运行爬虫
cd energy_storage && python3 crawler_multi.py

# 运行自愈检查
cd energy_storage && python3 self_healing_harness.py
```

## 六、常用操作

### 6.1 查看系统状态

```bash
# 查看定时任务列表
openclaw cron list

# 查看记忆摘要
cd memory && python3 context_loader.py

# 搜索记忆
cd memory && python3 summarizer.py search "关键词"

# 查看检索审计
cd memory && python3 retrieval_audit.py recent 10
```

### 6.2 修改配置

| 配置文件 | 路径 | 说明 |
|----------|------|------|
| Harness配置 | `harness/harness_config.yaml` | 约束规则、阈值 |
| 邮箱配置 | `email_config.json` | SMTP设置 |
| 爬虫配置 | `crawler_config.yaml` | 站点、关键词 |

### 6.3 关键文件位置

