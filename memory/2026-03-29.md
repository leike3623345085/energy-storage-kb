## IMA 知识库文件上传修复

**时间**: 2026-03-29 10:48  
**问题**: COS 上传返回 403 AccessDenied  
**解决**: 正确实现 COS 临时密钥签名算法

### 问题根因

COS 临时密钥上传需要正确计算 `Authorization` header，不能只用 `x-cos-security-token`。

### COS 签名算法

```python
def cos_sign(secret_id, secret_key, method, uri, host, content_type):
    # 1. KeyTime
    key_time = f"{start_time};{end_time}"
    
    # 2. SignKey
    sign_key = HMAC_SHA1(secret_key, key_time)
    
    # 3. HttpString (注意：URI 和 Header value 需要 URL encode)
    encoded_uri = quote(uri, safe="/")
    http_headers = "content-type=application%2Fpdf&host=..."
    http_string = f"{method.lower()}\n{encoded_uri}\n\n{http_headers}\n"
    
    # 4. StringToSign
    string_to_sign = f"sha1\n{key_time}\n{SHA1(http_string)}\n"
    
    # 5. Signature
    signature = HMAC_SHA1(sign_key, string_to_sign)
    
    # 6. Authorization
    auth = f"q-sign-algorithm=sha1&q-ak={secret_id}&q-sign-time={key_time}&q-key-time={key_time}&q-header-list=content-type;host&q-url-param-list=&q-signature={signature}"
```

### 关键修正

1. URI 需要 `quote(uri, safe="/")` URL encode
2. Header value（如 `application/pdf`）需要 encode 为 `application%2Fpdf`
3. `HttpString` 格式：`method\nuri\nparams\nheaders\n`（注意换行）

## 爬虫并行化原型实现

**时间**: 2026-03-29 16:20-16:25  
**目标**: 验证 HARNESS 自动任务拆解中的并行执行可行性

### 实现内容

| 文件 | 功能 | 代码行 |
|------|------|--------|
| `crawler_parallel.py` | 并行爬虫核心 | ~350 行 |
| `benchmark_crawler.py` | 性能对比测试 | ~150 行 |

### 核心改进

**串行 → 并行转换**:
```python
# 串行（现状）
for source in sources:
    result = crawl(source)  # 逐个执行

# 并行（优化后）
with ThreadPoolExecutor(max_workers=4) as executor:
    futures = [executor.submit(crawl, s) for s in sources]
    results = [f.result() for f in futures]  # 同时执行
```

### 性能测试结果

| 指标 | 串行 | 并行 (2 workers) | 提升 |
|------|------|------------------|------|
| 总耗时 | 0.17s | 0.09s | **1.9x** |
| 获取条数 | 106 条 | 106 条 | - |
| 时间节省 | - | 48% | - |

**数据源**: 北极星储能网 (94条) + 储能中国网 (12条)

### 扩展性预期

| 数据源数量 | 串行估算 | 并行估算 (4 workers) | 提升 |
|-----------|---------|---------------------|------|
| 2 个 | ~0.2s | ~0.1s | 2x |
| 4 个 | ~0.4s | ~0.15s | 2.7x |
| 8 个 | ~0.8s | ~0.25s | 3.2x |

### 关键特性

1. **错误隔离**: 单个数据源失败不影响其他源
2. **统一聚合**: 自动去重、合并结果
3. **性能统计**: 记录每个数据源的耗时和成功率
4. **配置化**: 数据源通过配置添加，无需改代码

### 与 HARNESS 的对应

### 1. XML 提示格式化 (XML Prompt Formatter)

**借鉴点**: GSD 的 XML 结构化提示

**核心特性**:
```xml
<task version="1.0" type="daily_report">
  <objective priority="high">基于 4 条资讯生成日报</objective>
  <context>
    <data_sources count="4">...</data_sources>
    <date>2026-03-29</date>
    <metadata>...</metadata>
  </context>
  <constraints>
    <constraint name="format" required="true">markdown</constraint>
  </constraints>
  <output_format type="markdown">
    <required_sections>...</required_sections>
  </output_format>
  <quality_criteria>...</quality_criteria>
</task>
```

**收益**:
- 结构化输入，AI 理解更精确
- 约束条件明确，减少偏差
- 质量标准可量化

### 2. 波式并行执行 (Wave-Based Parallelism)

**借鉴点**: GSD 的"波式并行性"(wave-based parallelism)

**执行模型**:
```
Wave 1: [爬虫A, 爬虫B, 爬虫C]  ← 无依赖，并行执行 (耗时 ~0.5s)
   ↓
Wave 2: [数据清洗]              ← 依赖 Wave 1 (耗时 ~0.1s)
   ↓
Wave 3: [报告生成]              ← 依赖 Wave 2 (耗时 ~0.1s)

总耗时: ~0.7s (vs 串行 ~1.7s)
提升: 2.4x
```

**核心特性**:
- 自动依赖解析
- 波次内并行，波次间串行
- 错误隔离（单任务失败不影响波次）
- 支持自动重试

**测试结果**:
- 3 个爬虫并行: 0.5s (vs 串行 1.5s)
- 效率提升: 3x

### 3. 原子化 Git 提交 (Atomic Git Commit)

**借鉴点**: GSD 的原子化 Git 提交理念

**提交格式**:
```
[CRAWL] 北极星储能网爬虫 ✅

成功获取 94 条新闻

详细变更:
  - 新增数据 94 条
  - 去重后有效 94 条
  - 耗时: 0.5s

文件变更:
  - data/crawler/bjx_20260329.json

元数据:
  source: 北极星
  items: 94
  duration: 0.5
```

**核心特性**:
- 任务级原子提交
- 结构化提交信息
- 自动元数据记录
- 失败回滚支持
- HARNESS 集成（on_crawl_complete/on_report_generated/on_sync_complete）

### 4. 与现有 HARNESS 架构的融合

| 输入 | XML 结构化提示 | `DailyReportXMLTemplate` |
