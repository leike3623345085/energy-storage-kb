## 2026-03-22

### 版本切换导致数据源断裂 (INC-20260322-001)
**问题**：日报生成器切换为 Harness v3 后，报告内容为空，201条爬虫数据未读取
**严重程度**：high
**根因**：
- 旧版 `generate_report_enhanced.py` 从 `crawler/*.json` 读取数据
- 新版 `generate_report_harness_v3.py` 只从 `news.jsonl` 读取
- `news.jsonl` 不存在，导致日报是"空壳"（有格式无内容）
- 置信度100/100（无数据需要核查），未触发警告

**为何未发现**：
- 报告格式完整、有热点标题、SWOT框架、产业链章节
- 视觉上正常，直到系统自愈监控报"数据不足"才发现

**修复**：
- 新增 `load_today_news()` 函数，支持多数据源读取
- 自动回退：如果 `news.jsonl` 为空，自动读取 `crawler/*.json`
- 数据量检查：少于10条时触发 Guardrails 警告

**教训**：
- **格式正确 ≠ 内容正确** —— 必须验证数据通路
- 版本切换时，必须检查新旧版本的数据源兼容性
- 空数据应该触发警告，而不是显示100%置信度

## 2026-03-14

### 定时任务超时自动处理
**问题**：3个定时任务连续超时（系统巡检/日记保存/工作日志保存）
**根因**：
1. self_healing.py 包含多个长时间子命令，累计超时
2. diary_save.py 调用 `openclaw sessions history` CLI 命令卡住
3. 超时设置不合理

**修复**：
1. 创建 self_healing_fast.py - 轻量版只检查不修复
2. 创建 diary_save_simple.py - 直接读取文件，不依赖 CLI
3. 调整超时：巡检60秒、日记30秒、工作日志120秒

**验证**：手动测试新脚本，执行时间 < 5秒

## 2026-03-11

### 工作原则确立
**问题**：执行任务时反复询问用户确认，降低效率
**根因**：过度谨慎导致决策延迟
**修复**：确立原则 — 遇到问题直接执行最优方案，事后通报结果

### 对话会话抢占
**问题**：对话说到一半突然无响应，心跳消息抢占主会话
**根因**：OpenClaw心跳每30秒触发一次，与正常对话共用会话
**修复**：创建独立守护进程 daemon_guardian.py 后台静默执行

### 任务执行中断
**问题**：Cron任务执行时Agent工具调用验证错误，邮件发送步骤被跳过
**根因**：Agent在isolated session中工具调用遇到验证错误
**修复**：部署自愈监控机制，增加多重保障

## 2026-03-06

### 日报生成任务执行失败
**问题**：定时任务状态显示"ok"，但实际报告未生成
**根因**：AGENT 收到指令后只解释了脚本功能，未实际调用 exec 工具
**修复**：修改任务指令，明确标注"必须使用 exec 工具实际执行命令"
**性质**：执行能力问题，非配置问题
