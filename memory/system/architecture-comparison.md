## 二、关键差距详解

### 🔴 P0 - 必须改进（影响核心能力）

#### 1. 自动任务拆解

**现状问题:**
- 复杂任务需要手动调用 `sessions_spawn`
- 用户需要明确说"分解这个任务"

**目标状态:**
```python
# 理想状态：自动识别并拆解
用户: "帮我分析储能行业并生成报告"
系统自动：
  1. spawn(数据收集子任务)
  2. spawn(分析报告子任务) 
  3. spawn(可视化子任务)
  4. 汇总结果
```

**实现思路:**
- 在任务接入阶段增加"复杂度评估"
- 超过阈值自动拆解
- 维护子任务依赖关系图

---

#### 2. 并行执行 Sub-Agents

**现状问题:**
- `sessions_spawn` 是串行的
- 多个独立子任务只能一个个执行
- 浪费时间

**目标状态:**
```python
# 并行执行独立子任务
parallel_spawn([
  {"task": "爬虫A站点", "agent": "crawler"},
  {"task": "爬虫B站点", "agent": "crawler"},
  {"task": "爬虫C站点", "agent": "crawler"}
])
```

**实现思路:**
- 识别任务间依赖关系
- 无依赖的任务并行执行
- 汇总所有结果后返回

---

#### 3. 漂移检测

**现状问题:**
- Agent 执行过程中可能偏离目标
- 没有自动检测和纠正机制
- 用户发现问题后才能手动纠正

**目标状态:**
```python
# 每步执行后检查
execute_step()
if drift_detected(current_result, expected_result):
    retry_with_correction()
    or
    escalate_to_user()
```

**实现思路:**
- 维护"预期结果"基准
- 每步执行后进行语义相似度检测
- 偏离时自动修正提示或通知用户

---

### 🟡 P1 - 建议改进（提升体验）

#### 4. 任务合法性预检

**现状问题:**
- 有些任务执行到一半才发现不可行
- 浪费时间和资源

**目标状态:**
```python
def task_validation(task):
    checks = [
        "权限检查：是否有权访问？",
        "可行性检查：工具是否支持？",
        "资源检查：文件/数据是否存在？"
    ]
    return all(checks)
```

---

#### 5. Docker 沙箱隔离

**现状问题:**
- 所有任务在主机直接执行
- 存在安全风险
- 不同 session 可能互相影响

**目标状态:**
- 每个子任务在独立 Docker 容器
- 文件系统隔离
- 可审计、可回滚

**实现难度:** 高（需要重构基础设施）

---

#### 6. 内置常用 Skills

**现状问题:**
- Skills 需要单独安装
- 没有统一的技能市场

**目标状态:**
- 预装常用技能（爬虫、报告生成、数据分析）
- 一键启用/禁用

---

### 🟢 P2 - 可选改进（锦上添花）

#### 7. 上下文自动压缩

**现状问题:**
- 长对话需要手动总结
- 可能爆上下文窗口

**目标状态:**
- 自动检测上下文长度
- 超过阈值自动压缩历史
- 保留关键信息

---

## 三、具体优化建议

### 短期（1-2周）

1. **实现 `parallel_spawn`**
   - 允许并行执行多个独立子任务
   - 提升多站点爬虫等场景效率

2. **增加漂移检测机制**
   - 在复杂任务中每步检查
   - 偏离目标时自动修正

### 中期（1-2月）

3. **自动任务拆解**
   - 基于任务复杂度自动判断是否需要拆解
   - 维护任务依赖关系图

4. **任务合法性预检**
   - 执行前检查权限、可行性、资源

### 长期（3-6月）

5. **Docker 沙箱（可选）**
   - 评估是否有必要
   - 当前单机部署可能不需要

**结论**: OpenClaw 在状态持久化和异常处理方面已对标 HARNESS，但在**自动拆解**、**并行执行**、**漂移检测**三个核心能力上存在明显差距，建议优先改进。
