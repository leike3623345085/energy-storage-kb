# DeerFlow 2.0 核心架构

> 来源: 智猩猩AI / 微信公众号
> 一个面向复杂任务处理的 Agent 框架

---

## 一、核心能力（5大模块）

### 1. Skills 与 Tools

**内置 Skills**:
- 研究 (Research)
- 报告生成
- 幻灯片制作
- 网页生成
- 图文视频创作

**特点**:
- 一键添加、替换、组合 skills
- 支持按需渐进加载，不浪费 token
- Tools 可插拔：网页搜索、抓取、文件操作、Bash执行
- 可通过 MCP Server 或 Python 函数无限扩展

---

### 2. Sub-Agents（子智能体）

复杂任务 DeerFlow 会进行**自动拆解**:
- 主智能体按需动态拉起子智能体
- 每个子智能体拥有**独立上下文**、工具和终止条件
- 可**并行执行**
- 最后由主智能体汇总结果

**优势**: 能处理从几分钟到几小时的任务

---

### 3. Sandbox 与文件系统

**给 Agent 一台"电脑"**:
- 每个任务在**隔离的 Docker 容器**里运行
- 拥有完整的文件系统: skills、workspace、uploads、outputs
- Agent 可以:
  - 读写和编辑文件
  - 执行 bash 命令和代码
  - 查看图片
- 整个过程都在 sandbox 内完成
- **可审计、会隔离**，不同 session 之间不会互相污染

---

### 4. Context Engineering

- 子智能体的**上下文完全隔离**，避免干扰
- DeerFlow 能够**自动总结、压缩、持久化**中间结果
- 超长任务不爆上下文，稳定跑完复杂流程

---

### 5. 长期记忆

- **跨会话积累**: 个人偏好、写作风格、技术栈
- memory 保存在**本地**
- **控制权始终在用户手里**

---

## 二、额外特性

- 支持 **Claude Code** 直接交互
- 内嵌 **Python Client**
- **多模型兼容**
- 集成智能搜索抓取工具 **InfoQuest**

---

## 三、与 OpenClaw 的对比

| DeerFlow 2.0 | OpenClaw |
|--------------|----------|
| Docker Sandbox | 主机直接执行 |
| 自动任务拆解 | 需手动 sessions_spawn |
| Context 自动压缩 | 手动 memory 管理 |
| 内置多种 Skills | Skills 需单独安装 |
| MCP Server 扩展 | 工具通过 Python 实现 |
