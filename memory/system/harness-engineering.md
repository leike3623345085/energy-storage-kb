# Agent Harness 标准工作流程

> 工业级 Harness 的标准执行流程
> 来源: 今日头条/豆包 AI 分享

---

## 一、Harness 标准执行流程（10步）

### 1. 任务接入
用户提出需求，Harness 先做**目标解析**、**合法性校验**。

### 2. 任务拆解与规划
把复杂任务拆成多步子任务，确定执行顺序。

### 3. 上下文初始化
构建系统提示、加载历史状态、分配工具权限。

### 4. 调度 Agent 执行
Harness 下发子任务，Agent 思考、调用工具、返回结果。

### 5. 结果校验与漂移检测
Harness 判断是否偏离目标、是否逻辑有效、是否需要重试。

### 6. 状态持久化
将进度、记忆、中间结果存入外部存储，不依赖上下文窗口。

### 7. 异常处理
失败自动重试、修正提示、切换策略、回滚状态。

### 8. 循环执行直到完成
一步步推进，直到所有子任务结束。

### 9. 结果汇总与交付
Harness 整合最终结果，返回给用户。

### 10. 数据沉淀
全流程日志保存，用于模型评测、提示优化、系统迭代。

## 二、Harness 本质定位

| 维度 | Agent Harness |
|------|---------------|
| **角色定位** | 控制系统 / 操作系统 |
| **基础循环** | 管理、监控、稳定、持久、调度 |
| **核心关注点** | 能不能稳定、长期、可靠地跑 |

**核心理念**: 整套流程全部由 Harness 主导，Agent 只负责执行。
